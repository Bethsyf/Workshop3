# My Merch

A continuación, se describen las consideraciones técnicas que se deben tener en cuenta para brindar solución al producto a entregar para con el Workshop de SPRINT # 3.

El trabajo se desarrollará en equipo máximo 3 personas.


# CONSIDERACIONES TÉCNICAS
- Crear un repositorio en GitHub, donde se evidencia las ramas una main (tendrá todo el proyecto), y 3 rama Dev-nombre del integrante del equipo.. se evidenciara el trabajo de forma individual por la cantidad de commit que haga al repositorio.

- Debe hacer uso de ReactJS como marco de trabajo JS - -https://es.reactjs.org/
- como marco de trabajo de CSS puede usar el deseado. (Libre)
- Debe permitir realizar el registro e inicio de sesión de distintos usuarios. Integrar módulo de autenticación haciendo uso de Firebase.
- Debe tener inicio de sesión con google y facebook
- Establecer enrutamiento dinámico con React-router.
- CRUD con firebase (agregar, listar, editar, eliminar, también buscar) (NO uso de API’s)
- Tener en cuenta las recomendaciones y estructura del README.md
profesional compartido en el SPRINT 3.
- Implementar Formik y Yup
- Uso de rutas públicas y privadas
- Publicar el resultado del frontend haciendo uso de Netlify, Vercel o GitHub Pages, Deploy con Firebase
- Cargar imágenes al proyecto por medio de ejecución de POST a un servidor
(cloudinary)
- Implementar el carro de compras
- Aplicar zoom en la imagen al hacer hover
